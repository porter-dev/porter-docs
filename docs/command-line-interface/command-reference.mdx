Here you'll find a reference for the most common Porter CLI commands:

- [`porter auth`](./porter-auth): contains commands for authenticating with a Porter instance.
- [`porter cluster`](./porter-cluster): contains commands for reading from a connected cluster.
- [`porter config`](./porter-config): contains commands for controlling local configuration settings.
- [`porter connect`](./porter-connect): contains commands for connecting to external clusters and providers.
- [`porter create`](./porter-create): creates a new application.
- [`porter delete`](./porter-delete): deletes a release.
- [`porter get`](./porter-get): fetches basic information about an application.
- [`porter job`](./porter-job): contains commands for interacting with jobs.
- [`porter list`](./porter-list): lists applications or jobs for the current cluster.
- [`porter logs`](./porter-logs): logs the output from a given application.
- [`porter project`](./porter-project): contains commands for interacting with a Porter project.
- [`porter registry`](./porter-registry): contains commands for interacting with a Docker registry in a project.
- [`porter run`](./porter-run): allows users to execute a command on a remote container.
- [`porter update`](./porter-update): builds and updates a specified application.

Here you'll find a reference for the most common Porter CLI commands:

- [`porter auth`](./command-reference/porter-auth): contains commands for authenticating with a Porter instance.
- [`porter cluster`](./command-reference/porter-cluster): contains commands for reading from a connected cluster.
- [`porter config`](./command-reference/porter-config): contains commands for controlling local configuration settings.
- [`porter connect`](./command-reference/porter-connect): contains commands for connecting to external clusters and providers.
- [`porter create`](./command-reference/porter-create): creates a new application.
- [`porter delete`](./command-reference/porter-delete): deletes a release.
- [`porter get`](./command-reference/porter-get): fetches basic information about an application.
- [`porter job`](./command-reference/porter-job): contains commands for interacting with jobs.
- [`porter list`](./command-reference/porter-list): lists applications or jobs for the current cluster.
- [`porter logs`](./command-reference/porter-logs): logs the output from a given application.
- [`porter project`](./command-reference/porter-project): contains commands for interacting with a Porter project.
- [`porter registry`](./command-reference/porter-registry): contains commands for interacting with a Docker registry in a project.
- [`porter run`](./command-reference/porter-run): allows users to execute a command on a remote container.
- [`porter update`](./command-reference/porter-update): builds and updates a specified application.

Here you'll find a reference for the most common Porter CLI commands:

- [`porter auth`](https://docs.porter.run/cli/command-reference/porter-auth): contains commands for authenticating with a Porter instance.
- [`porter cluster`](https://docs.porter.run/cli/command-reference/porter-cluster): contains commands for reading from a connected cluster.
- [`porter config`](https://docs.porter.run/cli/command-reference/porter-config): contains commands for controlling local configuration settings.
- [`porter connect`](https://docs.porter.run/cli/command-reference/porter-connect): contains commands for connecting to external clusters and providers.
- [`porter create`](https://docs.porter.run/cli/command-reference/porter-create): creates a new application.
- [`porter delete`](https://docs.porter.run/cli/command-reference/porter-delete): deletes a release.
- [`porter get`](https://docs.porter.run/cli/command-reference/porter-get): fetches basic information about an application.
- [`porter job`](https://docs.porter.run/cli/command-reference/porter-job): contains commands for interacting with jobs.
- [`porter list`](https://docs.porter.run/cli/command-reference/porter-list): lists applications or jobs for the current cluster.
- [`porter logs`](https://docs.porter.run/cli/command-reference/porter-logs): logs the output from a given application.
- [`porter project`](https://docs.porter.run/cli/command-reference/porter-project): contains commands for interacting with a Porter project.
- [`porter registry`](https://docs.porter.run/cli/command-reference/porter-registry): contains commands for interacting with a Docker registry in a project.
- [`porter run`](https://docs.porter.run/cli/command-reference/porter-run): allows users to execute a command on a remote container.
- [`porter update`](https://docs.porter.run/cli/command-reference/porter-update): builds and updates a specified application.
